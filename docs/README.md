我正在开发一个基于 Vue3 + Supabase 的可视化建站平台，网站结构由“导航栏区块、多个页面、页脚区块”组成，其中导航栏区块和页脚区块属于网站级配置，而页面是网站的内容单元。每个页面通过名称与路径自动生成导航菜单，路径结构支持多级嵌套，例如 /about 自动成为一级菜单，而 /about/contact 自动成为其二级菜单。
页面由多个“区块实例”组成。用户在页面编辑器中从左侧区块模板库拖拽模板到画布区域，即可创建区块实例。区块实例在创建时会完整拷贝区块模板的 html_code（采用 TailwindCSS + Handlebars）和 schema（基于 JSON Schema 用于描述字段与默认值），并独立存储，因此区块实例与其原始模板完全解耦：后续无论是区块实例的修改，还是模板的更新，都不会彼此影响。用户选中区块实例后，可以在右侧属性编辑面板中基于 schema 自动生成的可视化表单编辑内容和样式，实时更新页面效果。
3: 平台提供多网站管理能力，通过 GitHub OAuth 登录，将用户的 GitHub ID 持久化到数据库。在用户发布网站时，通过 GitHub API 在其 GitHub 账户下创建仓库，并将仓库 ID 存储到数据库，用于后续的部署和域名管理。平台支持一键发布到 GitHub Pages，支持绑定自定义域名，同时也支持在线预览整站或单页，并支持将全站导出为完整的静态资源（HTML/CSS/JS/图片等）。在数据结构设计中，网站和页面均需要支持 SEO 字段，如标题、描述和关键词。
4: 平台包含“模板市场”，支持三类模板：区块模板、页面模板、整站模板。用户可收藏任意模板，也可以通过整站模板一键克隆网站，通过页面模板快速创建页面，或者在编辑器左侧区块库中直接使用收藏的区块模板。同时，用户可以将自己创建的网站、页面或单个区块发布到模板市场，形成用户驱动的内容生态。
5: 在模板市场能力之上，平台集成了 AI 建站功能。AI 不负责生成网站样式，而是基于用户所选的整站模板，只自动生成各个页面内区块实例所需的数据内容部分（包括文字、标题、段落、图标、图片等）。换言之，AI 只负责填充内容，不改变模板本身的结构与样式。AI 生成内容后，会按照平台既有的数据结构自动创建网站、页面 and 区块实例，从而确保 AI 创建的网站与用户手工创建的网站具备同等可编辑性、可视化操作性与一致的发布流程。
