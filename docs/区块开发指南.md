# 区块模板开发指南 (Block Development Guide)

欢迎来到 Web Studio 区块开发指南。本指南旨在帮助开发者和设计师理解区块的核心架构，并学习如何创建高质量、可复用且高度可配置的区块模板。

---

## 1. 区块核心组成

一个区块模板由三部分核心定义组成：

1.  **HTML 结构 (`html_code`)**: 使用 **Handlebars** 语法编写的 HTML 片段。
2.  **样式声明**: 通过 **Tailwind CSS** 类名直接嵌入在 HTML 中。
3.  **配置协议 (`schema`)**: 使用 **JSON Schema** 定义的属性规范，驱动属性编辑面板。

---

## 2. HTML 编写规范 (Handlebars)

区块使用 Handlebars 进行数据绑定。所有在 `schema` 中定义的属性都可以通过 `{{ key }}` 的方式注入。

### 基础用法
```html
<h2 class="text-3xl font-bold">{{ title }}</h2>
<p>{{ description }}</p>
```

### 条件渲染
```html
{{#if showButton}}
  <button class="...">{{ buttonText }}</button>
{{/if}}
```

### 循环渲染 (例如列表)
```html
<ul class="space-y-2">
  {{#each features}}
    <li class="flex items-center gap-2">
      <span class="text-primary">✓</span> {{this}}
    </li>
  {{/each}}
</ul>
```

---

## 3. 样式开发规范 (Tailwind CSS)

本平台深度集成 Tailwind CSS。在编写区块时，请遵循以下原则：

*   **响应式优先**：使用 `md:`, `lg:` 前缀确保区块在各种屏幕下表现良好。
*   **暗黑模式支持**：虽然平台默认可能处于暗黑环境，但建议显式使用 `dark:` 类名（如果支持切换）。
*   **颜色变量**：尽量使用平台提供的语义化颜色，如 `bg-primary`, `text-secondary`。
*   **容器限制**：区块外层通常会被包裹在一个 `section` 或 `div` 中。建议在外层添加合适的 `py-` (vertical padding) 以保持页面节奏感。

## 4. 配置协议规范 (JSON Schema)

`schema` 决定了用户在右侧面板中能看到哪些配置选项。

### 字段类型与 UI 映射 (Field Mapping)

平台根据 Schema 的 `type` 和 `format` 属性自动选择最合适的 UI 组件：

| Schema 类型 | Format / 属性 | UI 组件 | 用途 |
| :--- | :--- | :--- | :--- |
| `string` | - | 文本输入框 | 短文本、标题 |
| `string` | `format: "textarea"` | 多行文本框 | 描述、正文 |
| `string` | `format: "color"` | 颜色选择器 | 背景色、文字颜色 |
| `string` | `format: "uri"` | 图片/资源地址 | 背景图、图标、外链 |
| `boolean` | - | 开关 (Switch) | 功能显隐切换 |
| `number` | - | 数字输入框 | 间距、字体大小、数量 |
| `array` | - | 列表编辑器 | 导航菜单、特性列表、FAQ |
| `string` | `enum: [...]` | 下拉选择器 | 预设样式、主题选择 |

### 核心设计原则

1.  **友好的标题 (`title`)**：始终提供 `title` 属性。如果缺失，系统将显示原始字段名（如 `buttonText`），这对非技术用户不友好。
2.  **明确的默认值 (`default`)**：每个字段都必须有合理的默认值。
3.  **详细的说明 (`description`)**：对于复杂的配置，使用 `description` 提供提示信息，它会以小字形式出现在输入框下方。
4.  **分类与逻辑分组**：建议将相似的属性（如“内容”和“样式”）分开排列，通过 `properties` 的定义顺序来控制面板的展示顺序。

### 进阶：对象与数组

#### 数组 (Arrays) - 循环物料
当需要用户配置列表（如轮播图、导航）时使用。建议定义 `items.type: "object"`。
```json
"features": {
  "type": "array",
  "title": "特性列表",
  "items": {
    "type": "object",
    "properties": {
      "icon": { "type": "string", "title": "图标" },
      "text": { "type": "string", "title": "描述" }
    }
  }
}
```

### 数据校验 (Validation)

利用 JSON Schema 的原生校验能力提高模板稳定性：
*   **必填项**：在根对象使用 `"required": ["title", "link"]`。
*   **范围限制**：对数字使用 `minimum`, `maximum`。
*   **格式检查**：对字符串使用 `pattern` (正则) 或内置 `format`。

### 示例 Schema
```json
{
  "type": "object",
  "properties": {
    "title": {
      "type": "string",
      "title": "标题文字",
      "default": "我们的优势"
    },
    "theme": {
      "type": "string",
      "title": "配色方案",
      "enum": ["light", "dark", "gradient"],
      "default": "light"
    },
    "showIcon": {
      "type": "boolean",
      "title": "显示图标",
      "default": true
    }
  },
  "required": ["title"]
}
```

## 5. 开发工作流

1.  **进入管理后台**：点击顶部导航栏的 **Templates**。
2.  **创建/编辑**：在左侧代码区编写 HTML 和 JSON Schema。
3.  **实时验证**：
    *   在右下角的 **Test Properties** 面板中输入模拟数据。
    *   在右上角的 **Live Preview** 区查看渲染结果。
4.  **格式化**：点击编辑器右上角的 **Format** 按钮（或快捷键 `Alt+Shift+F`）整理代码。
5.  **保存并使用**：保存后，该区块将出现在所有页面编辑器的“区块库”中。

---

## 6. 最佳实践与建议

*   **完全解耦设计**：一旦区块被拖入页面，它就会变成一个独立的“实例”。这意味着你的模板修改不会影响已经存在的页面，因此在修改模板时请大胆尝试。
*   **默认值很重要**：为每个属性设置合理的 `default` 值，确保用户刚拖出区块时就有良好的视觉呈现。
*   **注释与语义化**：虽然 HTML 片段很小，但良好的注释有助于以后维护复杂的逻辑。
*   **性能优化**：避免在区块中使用过大的内联资源（如 Base64 图片），尽量通过 URL 引用或 `props` 传入。
